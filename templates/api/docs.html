DocShift API Reference

Integrate powerful document conversion and translation directly into your application using our REST API.

1. Authentication

Authenticate your requests by sending your API Key in the Authorization header as a Bearer token. 

Authorization: Bearer YOUR_API_KEY

Get your API key →

2. Convert Documents

Submit files for conversion using a multipart/form-data POST request.

POST /api/v1/convert/{tool_slug}/

Parameters

  • tool_slug (url): The tool you wish to run (e.g., merge_pdf, word_to_pdf)
  • file (form-data): The file to convert. Max size: 50MB.

Example Request (cURL)

curl -X POST https://yourdomain.com/api/v1/convert/pdf_to_images/ \
  -H "Authorization: Bearer YOUR_API_KEY" \
  -F "file=@/path/to/document.pdf"

3. Translate Documents

Translate entire .docx files while preserving formatting.

POST /api/v1/translate/

Parameters

  • file (form-data): The .docx file to translate.
  • source_lang (form-data, optional): Source language code. Defaults to `auto`.
  • target_lang (form-data, optional): Target language code. Defaults to `en`.

Example Request (Python)

import requests

url = "https://yourdomain.com/api/v1/translate/"
headers = {"Authorization": "Bearer YOUR_API_KEY"}
files = {"file": open("report.docx", "rb")}
data = {"target_lang": "es"}

response = requests.post(url, headers=headers, files=files, data=data)
print(response.json())

4. Webhooks (Callbacks)

Because document conversion and translation can take time, the API returns a 202 Accepted status immediately. Configure a Webhook URL in your dashboard to have DocShift automatically POST the results to your server when finished.

Webhook Payload

{
  "job_id": "b32a9e22-8df3-4c55-b44c-1b77c8e9",
  "status": "done",
  "download_url": "/job/b32a9.../download/",
  "error": null
}
